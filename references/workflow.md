# 中文技术博客协作工作流

## 模式边界

- **多轮共创**：用于从零写作和结构重构，与用户共同确认任务、标题、结构和章节注释。
- **全文润色**：保留标题、结构、技术立场和认知强度，只改善表达、节奏、术语和排版。
- **局部修改**：凡用户明确圈定有限范围，均保持范围外内容不变。
- **事实核查**：验证文章中的外部可核实主张；默认只核查，只有用户要求时才同步改正文。
- **定稿**：主 agent 直接完成最终修订和交付，不等同于独立审稿。
- **发布前检查**：检查发布阻塞项，不等同于独立审稿。
- **独立审稿**：仅在用户明确要求独立或多视角 reviewer 时启动。

## 从零写作

### 第一轮：任务与素材

1. 根据已知信息起草任务卡，不要求用户从空白表格开始填写。
2. 讨论目标读者的触发场景、已有知识、此前尝试、限制和读完后的可观察行为。
3. 盘点作者独有材料、可引用来源、可运行代码、数据、案例和无法公开的信息。
4. 区分用户提供的事实、作者经历、待验证主张、合理推断和仅用于说明的示例。
5. 明确发布平台、长度、格式和希望保留的作者声音。

### 第二轮：选题与标题

1. 形成文章的一句话承诺、不覆盖范围和放弃条件。
2. 标题未确定时提供 3–5 个候选，说明读者承诺、范围、差异和风险。
3. 标题已确定时检验其是否可兑现；除非标题明显误导或用户要求，不强制生成替代标题。
4. 等待用户确认标题和文章承诺，或按用户明确指示继续。

### 第三轮：结构与章节注释

1. 确定主导读者意图，并按需使用 Diátaxis 和扩展模式进行诊断。
2. 建立中心主张、分论点、证据、推理依据、反例、边界和置信度组成的论证图。
3. 先自由列要点，再组织为多级大纲。
4. 为每节写简短注释：读者问题、章节目的、必须讲清的信息、证据或代码、例子或反例、风险边界和过渡。
5. 与用户确认结构；有重大分歧时先修改结构，不提前写完整正文。

### 第四轮：正文与修订

1. 明确询问是否需要 TLDR，并确认整篇撰写还是分章节交付。
2. 按用户选择整篇撰写或分章节交付。
3. 不把合理但未提供的事故症状、指标、排查过程和团队行动写成作者亲历事实。
4. 先完成结构和论证，再处理语言、排版和发布细节。
5. 写完后依次检查结构承诺、证据与因果、教学可执行性、作者声音和发布质量。

用户明确要求跳过讨论、直接写作时，内部完成必要规划，只交付正文、明确假设和待核实项。

## 结构重构或重写

1. 盘点原文标题、读者假设、结构、核心主张、证据、代码、开头、结尾和作者声音。
2. 标记必须保留的事实、经历、判断、不确定性、术语和习惯表达。
3. 与用户补齐任务卡。
4. 提出推荐重构方向；只有确实存在高影响取舍时才提供多个方案。
5. 与用户讨论标题、大纲和章节注释，明确询问 TLDR 和交付方式，确认后再重写。
6. 重写不得把经验升级为普遍规律，也不得补造作者未提供的第一人称细节。

## 全文校对或语言润色

- 保持标题、结构、技术观点、例子、引用和结论不变，除非用户授权调整。
- 保持“可能、通常、观察到、感觉、在当前样本中”等认知强度，不改成更确定或更普遍的表达。
- 发现明显技术错误、逻辑断裂或危险建议时，不静默改动；按用户要求标注问题或单独列出。
- 默认直接交付润色稿；用户要求时再提供变更摘要或对照说明。

## 局部修改

- 以用户授权范围而非内容类型判断是否属于局部任务；章节、代码块、表格和多个指定片段也可采用局部模式。
- 从上下文推断读者和语气，只在技术立场、术语层级、修改边界或预期长度无法判断时提问。
- 不主动讨论全文结构或 TLDR，不启动独立 reviewer，不修改范围外内容。
- 微小修改直接交付替换文本；重大或非显而易见的修改简要说明目的。

## 事实核查

- **只核查**：默认输出结论、关键来源、适用条件、风险和建议措辞，不修改原文。
- **核查并修改**：用户明确要求时，先记录核查结果，再修改对应主张。
- 全文核查优先处理影响结论、安全、成本、决策和可执行性的高风险主张，不以篇幅换取虚假的全覆盖。

## 定稿、发布检查与独立审稿

- **定稿**：主 agent 完成必要修订，交付干净终稿、待核实项和必要变更摘要。
- **发布前检查**：检查标题承诺、链接、引用、代码、图片、敏感信息、版本和未解决风险；不自动 fan-out。
- **独立审稿**：只有用户明确要求时才按审稿编排执行；审稿结果不能替代事实核查。

## 交互原则

- 多轮共创要提供可讨论的具体草案，不把问题列表全部抛给用户。
- 将相关问题合并为一轮，解释每个高影响选择会改变什么。
- 用户确认过的标题、结构和范围形成当前基线；后续修改如需偏离，先说明原因并再次确认。
- 不为满足模板而增加无助于读者的章节、总结、案例或行动号召。
